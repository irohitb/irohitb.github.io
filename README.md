# Rohit Bhatia — Portfolio

Static portfolio site built with **Astro** (+ small vanilla scripts and one
React island for the theme toggle). Live at **[rohitbhatia.com](https://rohitbhatia.com)**
(GitHub Pages).

All content lives in `src/data/*.yml`. Each YAML file is read by a sibling typed
accessor (`src/data/<name>.ts`) via `loadYaml()`, so components never touch the
data format — editing a `.yml` file is all it takes to change the site.

## Develop

```bash
npm install
npm run dev      # http://localhost:4321
npm run build    # regenerate data (health + github) then astro build -> dist/
npm run preview
```

Data-only regen scripts:

```bash
npm run build:health   # Apple Health export -> src/data/health.yml
npm run build:github   # GitHub contributions (public API) -> src/data/github.yml
npm run build:resume   # Google Doc -> public/resume.pdf
```

## Data model

Everything is YAML, updated one of three ways: **hand-edited**, **generated by a
script**, or **computed at build time**.

| Section | File | How it updates |
| --- | --- | --- |
| Identity, links, hero lines, agent brief | `profile.yml` | Hand-edited |
| Work history | `work.yml` | Hand-edited |
| Recommendations (short + full) | `recommendations.yml` | Hand-edited |
| Music (now playing + favourites) | `music.yml` | Hand-edited |
| Reading / movies | `offTheClock.yml` | Hand-edited |
| Birthday age + countdown | `life.yml` (DOB) | Computed at build |
| GitHub contributions | `github.yml` | **Live** — token-free public API, hourly |
| Apple Health (workouts/kcal/steps) | `health.yml` | Generated from an export (see below) |
| Résumé PDF | `public/resume.pdf` | Generated from a Google Doc (see below) |

### GitHub contributions (live, no token)

`build-github.mjs` reads the username from `profile.yml` and fetches the rolling
last-365-days total from a public contributions API — **no PAT / secret needed**.
The `deploy` workflow runs it hourly, so the live count self-refreshes.

### Apple Health

There's no cloud API for Apple Health, so the pipeline is manual-drop:

1. Export from your phone (Health Export Kit JSON, Health Auto Export JSON, or
   the native `export.zip`) into `data/health-export/`.
2. `npm run build:health` parses it → `src/data/health.yml`.
3. Commit `health.yml` and push.

Raw exports are **git-ignored** (they contain granular personal data) — only the
3-number summary in `health.yml` is committed. See
[`data/health-export/README.md`](data/health-export/README.md).

### Résumé

The `resume` workflow (manual dispatch) exports the Google Doc configured in
`profile.yml` (`resume.sourceDocId`) to PDF and opens a PR with the refreshed
`public/resume.pdf`. The Doc must be shared "anyone with the link can view".

## Human / Agent view

A **Human / Agent** toggle in the nav swaps the whole page:

- **Human** — the normal site.
- **Agent** — a plain-text / markdown rendering (no images or UI, colored
  headings only), for AI agents. It explicitly states availability that the
  human UI doesn't.

The same brief is also served machine-readable at:

- **[`/agent`](https://rohitbhatia.com/agent)** — text-only page
- **[`/llms.txt`](https://rohitbhatia.com/llms.txt)** — the `llms.txt` convention

## Deploy

`.github/workflows/deploy.yml` builds and publishes to GitHub Pages on **push to
`main`**, **hourly** (to refresh the GitHub count), and **on demand**. It runs
`npm run build`, so each deploy ships fresh data. The custom domain is pinned via
`public/CNAME`.

> Pages **Source** must be set to **GitHub Actions** (Settings → Pages), not
> "Deploy from a branch".

## Conventions

- **Data is never hardcoded in components** — it all lives in `src/data/*.yml`
  behind typed accessors. See [`CLAUDE.md`](CLAUDE.md) for the full rules.
- **Orange (`--accent`) is reserved for section headings.** Other emphasis uses
  white/silver shades; theme values come from CSS variables in
  `src/styles/global.css` (`data-theme` on `<html>`).
- **Minimal client JS** — Astro components are static by default; only the theme
  toggle is a React island, and the typing / view-toggle / copy behaviours are
  tiny vanilla scripts.

## Structure

```
portfolio/
  src/
    data/         # *.yml content + typed accessors (*.ts) + loadYaml helper
    components/   # presentational .astro + AgentView + ThemeToggle island
    layouts/      # Base.astro (head, fonts, theme/view bootstrap)
    styles/       # global.css (theme tokens, responsive rules)
    pages/        # index.astro, agent.astro, llms.txt.ts
  public/         # assets, resume.pdf, CNAME
  scripts/        # build-health / build-github / build-resume
  data/
    health-export/  # drop Apple Health exports here (git-ignored)
  .github/workflows/ # deploy, resume
```
